# VibeStick 0.1.6 集成更新记录

- 发布日期：2026-07-15
- 适用设备：M5Stack StickS3
- 配套 Bridge：0.1.6
- Windows 远程输入协议：兼容客户端 1.14.2 及以上

## 更新目标

这一版把 StickS3 从“录完再上传”的语音遥控器，改造成低延迟的局域网流式语音输入终端，并补齐用量显示、任务提醒、电源管理与 Windows 远程输入链路。典型链路如下：

```text
StickS3 麦克风
  -> Unraid 上的 VibeStick Bridge / FunASR
  -> Windows 输入中继
  -> 当前光标所在应用
```

API Token、ASR Key 和账户凭据只保存在 Bridge/服务器环境变量中，不编译进固件，也不写入仓库。

## StickS3 操作方式

- 正面键单击：唤醒屏幕并向当前电脑输入发送 Enter。
- 正面键长按：开始流式语音输入；松开后立即停止采集并执行尾段校准。
- 正面键双击：主动刷新 Codex/Claude 用量；屏幕刚被唤醒时第一次操作只用于唤醒。
- 侧键单击：切换 Codex/Claude，并立即刷新当前账户与剩余额度。
- 拿起设备：通过 BMI270 运动检测自动亮屏。
- 翻转放置：自动关闭屏幕背光；拿起后恢复显示。

## 主要变化

### 1. 局域网流式 ASR

- StickS3 录音期间持续分块上传 PCM，不再等整段录音结束后才提交。
- Bridge 新增 FunASR 流式会话、热词读取和结果分段处理。
- 松开按键后，音频上传和最终校准进入独立任务，避免界面长期卡在“正在聆听”。
- 增加 GPIO 松开检测、断流回收、最长录音和事件队列保护，降低无线抖动导致的录音卡死概率。
- 稳定流式模式锁定已确认前文，只校正当前活动尾段，避免整段文字反复回退、重复输入或被最终结果覆盖。
- 增加静音/噪声门控、尾部幻觉过滤、英文开发术语和网址标准化能力。

### 2. Windows 输入中继

- Bridge 增加出站输入中继协议，可把 S3 的临时结果和最终结果发送到 Windows 当前光标位置。
- 支持本机与远程桌面副客户端架构，不要求修改 RDP 配置。
- 输入校正采用稳定尾段替换，保留已经锁定的长文本前文。
- Windows 客户端源码不在本仓库；本仓库保存协议端、Bridge 端和自动化测试。

### 3. Codex / Claude 用量与账户名

- Bridge 在服务器端轮询 Sub2-Usage，读取 5 小时、7 天剩余百分比和 `account_name`。
- Token 只由服务器环境变量提供；StickS3 只接收整理后的状态快照。
- 固件首页用账户名替代原来的“在线/离线”文字，并保留最近一次成功结果作为短时缓存。
- 双击正面键或单击侧键切换服务商时可主动刷新。

### 4. 按键发送与任务提示音

- 恢复原固件正面键单击发送 Enter 的行为；屏幕熄灭时也不会再吞掉这次发送操作。
- 恢复 Codex/Claude 完成、需要确认和错误三类提示音。
- 提示音在录音期间延后播放，只有播放成功后才消费事件，避免音频编解码器忙时漏报。
- Bridge 可接收 Windows 客户端上报的任务状态，再经现有状态接口传给 StickS3。

### 5. 电源与屏幕管理

- 接入 BMI270 IMU，用于翻转熄屏、移动唤醒和拿起亮屏。
- 电池供电且连续 60 分钟无操作时自动关机；录音中、任务运行中、USB 供电或充电时不触发。
- 关机前停止音频、IMU 和 Wi-Fi，释放扬声器、LCD 与传感器电源轨。
- 清除原固件的 LDO 保持状态，再向 M5PM1 PMIC 发送关机命令，避免“屏幕黑了但外设仍耗电”。
- 如果 PMIC 未能完全切断主电源，ESP32-S3 自动进入深度睡眠作为低功耗兜底。

### 6. 部署与维护

- 增加 Unraid Docker 模板、启动脚本和 supervisor，使 FunASR、Bridge 与辅助接口可作为一个容器维护。
- 增加 Windows PowerShell 安装/开发脚本。
- 固件使用更大的单应用分区，以容纳流式、IMU、中文字体与电源管理代码。
- 中文字体子集补充账户名所需字形；自定义音源只记录来源说明，不把第三方原声音频提交到仓库。

## 配置摘要

Bridge 端按需配置：

```env
VIBE_STICK_ASR_PROVIDER=local-funasr
VIBE_STICK_ASR_BASE_URL=ws://<funasr-host>:10095
VIBE_STICK_SUB2_USAGE_TOKEN=<server-only-token>
VIBE_STICK_SUB2_USAGE_BASE_URL=https://88api.ai/sub2-usage
VIBE_STICK_SUB2_CODEX_ACCOUNT_ID=<account-id>
VIBE_STICK_SUB2_CLAUDE_ACCOUNT_ID=<account-id>
```

真实值应写入未跟踪的 `.env` 或 Unraid Docker 变量。不要把 Token、Wi-Fi 密码或 API Key 提交到 Git。

## 已知限制

- 当前固件应用分区余量约 34.2%，后续新增字体或大型资源前仍应评估空间。
- “最终校准不改写已锁定前文”优先保证输入稳定；很早以前的错字应通过热词/标准化规则在流式阶段解决。
- 真正断电仍受硬件 PMIC 与电池板设计约束；本版同时提供 PMIC 关机和深度睡眠兜底。
- Windows 图形客户端属于配套项目，本仓库只包含与其通信的服务端协议实现。
